# Issue File Format

When creating files in `issues/`:

- Use the filename format: `ISSUE-XXX-short-description.md`
- Each issue file must include:
  - **Source**: which task, review, or triage produced this issue
  - **Description**: what the problem or risk is
  - **Priority**: low / medium / high / critical
  - **Status**: open / in-progress / resolved
- Link back to the originating task or story ID when applicable.
- Created by `/follow-up-triage` or manually — either way, same format.
