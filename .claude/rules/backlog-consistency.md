# Backlog Consistency

When editing `docs/backlog.md`:

- Every story must use the `US-XXX` ID format with a sequential number.
- Stories must include: title, "As a / I want / So that" block, and acceptance criteria.
- When a story moves to "In Progress", it must also be activated in `tasks/current.md`.
- When a story is completed, move it to a "## Completed" section (do not delete it).
- Never have more than one story "In Progress" unless the user explicitly requests
  parallel work.
- Keep story IDs stable — never renumber existing stories.
