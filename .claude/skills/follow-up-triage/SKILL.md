# Follow-Up Triage

Use this skill when residual risk or debt should not disappear into archived task notes.

## Workflow

1. Read `tasks/current.md`, especially:
   - `## Review Findings`
   - `## Understanding Remediation`
   - `## Override / Audit Note`
   - `## Notes`
2. Identify items that were:
   - accepted
   - deferred
   - left unresolved
3. Turn those items into visible follow-up work:
   - backlog stories in `docs/backlog.md`
   - issue entries in `issues/`
4. Record links or references in the task note so the debt is traceable.

## Rules

- Do not create duplicate follow-up items if equivalent work already exists.
- Keep follow-up items specific, actionable, and tied to a concrete risk or gap.
